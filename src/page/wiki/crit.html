开发规范

命名

  • 变量名

    驼峰，下划线都可以

    • 变量

      s: 字符串 例如：s_html,sHtml

      n: 数字 例如：n_page,nPage

      is: 逻辑 例如：is_first

      reg: 正则 例如：reg_email

      g: 全局 例如：g_count

    • 常量

      例如：const MAX_COUNT=20; 大写，下划线连接

    • 引用赋值变量

      如果引用对象是一个类 那变量名 最好大写开头 例如 const Transform = require('Transform');

      变量声明 一般小写开头 例如 const myName="xixi";

  • 函数名

    • _[方法名] 私有属性下划线开头
    • 单词尽量简写，少用拼音单词混用。尽量表达出函数作用

    • 动名组合：

      [动词_]名词[_行为] ， 如：get_name_byId , getNameById

  • 文件命名

  • 模块组件命名

    • 单词数量，不要超过3个
    • [动词_]名词[_状态] ， 如：set_title , set_title_hidden 或者 hide_title

  • 接口名字

  • class 类名
注释规范

    /**
     * 注释demo 描述
     * @function demo
     * @param {String} name string 类型参数
     * @param {String} type string 类型参数
     *        [p2=test1] 当参数=test1 时候描述
     *        [p2=test2] 当参数=test2 时候描述
     * @param {Object} obj 对象类型参数
     *        [obj.a=true] obj 中 a 属性描述
     * @constructor
     *
     **/
    function demo (name, type, obj) {
        /**
         * demo 中的 myname 属性
         * @property myname
         * @type String
         **/
        this.myname = name;

        /**
         * demo 中的 _mytype 属性
         * @property _mytype
         * @type String
         * @protected
         **/
        this._mytype = type;
    }

    /**
     * demo 中的 getNick 方法
     * @method  getNick
     * @param {String} name demo 的名字
     * @return {String} 返回昵称
     **/
    demo.prototype.getNick = function (name) {
        return "nick_" + this.name;
    }
